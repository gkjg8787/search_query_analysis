以下のJSONデータは、WebサイトのHTML構造を解析用に圧縮したものです。各キーの意味は以下の通りです。

基本構造:
    t: タグ名 (Tag name)
    ch: 子要素のリスト、または要素内のテキスト (Children or Text)

属性 (Attributes):
    i: ID属性 (id)
    c: クラス属性 (class)
    n: ネーム属性 (name) — フォーム送信時のパラメータ名
    tp: タイプ属性 (type) — text, submit, checkbox, radio 等の判別用
    ph: プレースホルダー (placeholder) — 入力欄のヒントテキスト
    v: 値 (value) — 選択肢や入力済みの値
    al: アクセシビリティラベル (aria-label) — 視覚的でない要素の役割説明
    r: ロール属性 (role) — search, button 等の定義
    h: リンク先URL (href)

# タスク
- この構造を解析し、以下のCSSセレクタを特定してください。
  - 検索キーワード入力ボックス
  - 検索実行ボタン
  - 検索オプション（カテゴリー選択、チェックボックス等）があればその種別とセレクタ

- セレクタは、できるだけユニークで堅牢なもの（ID優先、なければ属性セレクタやクラスの組み合わせ）を出力してください。また、カテゴリー選択などが select なのか input[type='checkbox'] なのかも明記してください。

# input
- サイトのHTMLを圧縮したJSON

# output
- json形式
```
{{
  "search_input_box": "#example_selector",
  "search_button":".example_class",
  "search_options":{
    "category":{"tag_type":"select", "query_name":"cate","options":[{"value":1, "text":"全商品"}, {"value":2, "text":"食品"}]}
  },
  "error_msg": "error values"
}}
```

- "search_input_box" : "あればCSSセレクタ、なければ空欄"
- "search_button" : "あればCSSセレクタ、なければ空欄"
- "search_options": "検索オプション。なければoutputに含めなく良い。カテゴリのみ見つけて。"
- "error_msg" : "search_input_box または search_buttonが見つからない場合の理由。 search_optionsが見つからない場合はなくて良い"

# 留意
- HTML(JSON)が必ずしも検索ボックスが出ているとは限らない。javascriptの描写待ち、ボット対策画面、検索とは無関係のHTML(JSON)の可能性あり。
